## 2024-04-06 - Dart Object Allocation Overhead in Bot AI
**Learning:** In Dart (specifically for Flutter games using offline bot controllers), methods returning standard unmodifiable structures (like a full card deck) can become significant bottlenecks if they reallocate and rebuild collections dynamically inside tight AI loops. `GameCard.fullDeck()` was rebuilding a 32-card Set on every call, which was frequently invoked by `CardTracker` to calculate `trumpsRemaining` and `isSuitExhausted` during bot decision-making. Since bot decision trees require hundreds or thousands of evaluations per turn, the garbage collection and collection-building overhead added up considerably.
**Action:** Always cache immutable, globally shared static datasets (like a complete standard deck of cards) using `Set.unmodifiable` attached to a static variable instead of rebuilding them per-call, especially when accessed by AI tracking algorithms.
